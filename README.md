# Clinic Cardiac

一个用于日常临床学习的轻量 Markdown / Obsidian 仓库。

目标只有三件事：
- 每天能快速记录
- 重要内容能逐步沉淀
- 复习入口足够简单，能坚持用

## 推荐结构

```text
.
|-- 00-首页.md
|-- 01-每日/
|-- 02-专题/
|-- 03-复习/
|-- 模板/
`-- 文档/
```

## 每个目录只做一件事

- `00-首页.md`
  - 仓库总入口
  - 放今天要看什么、常用链接、简单工作流

- `01-每日/`
  - 每天的学习记录、查房记录、临时问题
  - 原则：先记下来，不要求完美

- `02-专题/`
  - 已经想清楚、以后会反复看的内容
  - 例如：`[[胸痛]]`、`[[ACS]]`、`[[房颤抗凝]]`、`[[股动静脉穿刺]]`

- `03-复习/`
  - 放需要反复回看的东西
  - 只保留两个重点：`[[复习清单]]` 和 `[[错题与易错点]]`

- `模板/`
  - 只保留最常用的模板
  - 建议先保留：每日模板、专题模板

- `文档/`
  - 原始 PDF、制度文件、整理后的长笔记

## 简单工作流

1. 每天把新学到的东西先写进 `01-每日/`
2. 当某个知识点值得长期保留，再整理进 `02-专题/`
3. 容易忘、容易混、容易错的内容，放进 `03-复习/`
4. 原始资料和长篇整理稿，放进 `文档/`

## 为什么这样设计

- 目录少，打开仓库不会分心
- 每日记录和长期知识分开，后面整理更轻松
- 复习单独放一层，方便你每天固定看
- 先不做复杂数据库和脚本，避免维护成本高于收益

## 不建议现在就做的事

- 不要过早拆 `data/`、`helpers/`、`scripts/`
- 不要一开始就做太细的亚专科目录
- 不要把每天的临时想法直接写进专题笔记

## 使用建议

- `01-每日/` 解决“今天学了什么”
- `02-专题/` 解决“这件事以后怎么快速回顾”
- `03-复习/` 解决“什么东西我总是会忘”

涉及药物剂量、流程、禁忌证、围术期处理等内容时，实际临床请始终以最新指南、药品说明书、本院制度和上级医师意见为准。
